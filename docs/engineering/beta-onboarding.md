# Beta Onboarding Guide

Owner: Product + Engineering  
Last reviewed: 2026-04-15

## Purpose

This guide helps beta clients start testing quickly and report useful feedback.

## Prerequisites

- You received account approval from the admin team.
- You can sign in with Google or Email/Password.
- You have at least one bank statement PDF to test.

## 15-minute quickstart

1. Open the app home page and sign in.
2. Upload one statement on the home page.
3. Keep OCR enabled unless instructed otherwise.
4. Review extracted rows on the Review page.
5. Edit incorrect rows and export Excel.
6. Open Billing page and confirm usage changed.

## First-run smoke test

Use this checklist after first access:

- Login succeeds and page loads normally.
- PDF preview renders on the Review page.
- You can select a row and see matching PDF highlight.
- You can edit one cell and save.
- Excel export downloads and opens.
- Billing page shows updated document totals.

## Expected behavior

- OCR mode is generally more reliable for difficult layouts.
- Review page allows correction before final export.
- Billing is estimated live and finalized at month end.

## Reporting feedback

When reporting a bug, include:

- Your email/account
- Date/time (local timezone)
- Bank type and OCR on/off
- Session ID (from review page)
- Screenshot/error text
- What you expected vs what happened
