# Support and Escalation

Owner: Operations  
Last reviewed: 2026-04-15

## Support channels

- Primary: product support email/chat (internal team-managed)
- Secondary: admin contact via approved business channel

## Severity levels

- S1 Critical: service unavailable, data loss risk, login outage
- S2 Major: core workflow degraded, no clear workaround
- S3 Minor: cosmetic/UI issue, workaround exists

## Target response expectations

- S1: acknowledge within 1 hour, updates every 1-2 hours
- S2: acknowledge within 4 business hours, daily updates
- S3: acknowledge within 2 business days, triaged into backlog

## Escalation path

1. Support triage and reproduction
2. Engineering owner assigned
3. Product informed for client communication
4. Incident channel if S1/S2

## Required bug report fields

- User email/account id
- Endpoint/page where issue occurred
- OCR enabled or disabled
- Session ID (if review flow)
- Timestamp and timezone
- Screenshot and exact error text
